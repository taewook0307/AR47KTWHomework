자료구조
 : 특정 객체나 특정 단위의 메모리를 효율적으로 관리하는
   구조, 클래스, 함수, 알고리즘 등을 통틀어서 칭하는 것

자료구조는 개념
  자료를 이렇게 보관했더니 더 빠르게 찾고 더 많이 저장하고 더 쉽게 사용할 수 있다.


자료구조의 2가지의 데이터형
	1. 배열형
	2. 노드형

자료구조의 3종류의 구분
	1. 시퀸스 : 데이터를 순회했을 때 내가 입력한 순서대로 출력되는 것
	2. 연관 : 데이터를 순회했을 때 내가 입력한 순서대로 출력이 안 되고 순서가 어떠한 형태로든 변하여 출력되는 것
	3. 어뎁터

vector => 배열형 시퀸스 자료구조
list => 노드형 시퀸스 자료구조
map => 노드형 연관 자료구조

자료구조를 보관하는 모든 개념을 종합해서 컨테이너라고 부르기도 함.
vector => 배열형 시퀸스 컨테이너
list => 노드형 시퀸스 컨테이너
map => 노드형 연관 컨테이너


자료구조의 공통 인터페이스

  자료를 추가하는 인터페이스
  ex) vector : push_back, resize
  
  자료를 삭제하는 인터페이스
  
  자료를 순회하는 인터페이스
  ex) vector : size이용해서 for문


map의 문제점
편향트리 문제 : 이를 해결하기 위해 다양한 알고리즘이 만들어짐
map은 여러 종류의 스핀(맵을 회전시켜서 루트를 변경하여 균형을 어느정도 잡아주는 알고리즘) 알고리즘 중에 레드블랙 알고리즘을 사용
